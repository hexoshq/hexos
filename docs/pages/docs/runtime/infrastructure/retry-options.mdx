Options for configuring exponential backoff retry behavior.

Controls the number of attempts, delay progression, and error classification for
[retryWithBackoff](/docs/runtime/infrastructure/retry-with-backoff#retrywithbackoff). The optional `shouldRetry` predicate allows overriding the
default [isRetryableError](/docs/runtime/infrastructure/is-retryable-error#isretryableerror) logic for custom error handling.

Related: [NormalizedRetryConfig](/docs/runtime/infrastructure/normalized-retry-config#normalizedretryconfig) provides the runtime defaults,
[RetryConfig](/docs/common/infrastructure/retry-config#retryconfig) is the user-facing configuration.

```ts
interface RetryOptions {
    maxAttempts: number;
    initialDelayMs: number;
    maxDelayMs: number;
    multiplier: number;
    jitter: boolean;
    shouldRetry?: (error: unknown) => boolean;
}
```
