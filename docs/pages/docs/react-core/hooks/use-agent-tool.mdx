Hook that registers a frontend tool the agent can call for browser-side execution.

Frontend tools execute in the browser rather than on the server, providing access
to browser APIs (clipboard, geolocation, DOM, etc.). When the LLM invokes a tool
whose name matches a registered frontend tool, [useAgent](/docs/react-core/hooks/use-agent#useagent) detects the match
in [frontendToolsAtom](/docs/react-core/state-management/frontend-tools-atom#frontendtoolsatom) and executes it locally.

The tool is automatically unregistered when the component unmounts.

Related: [FrontendToolDefinition](/docs/react-core/tools/frontend-tool-definition#frontendtooldefinition) defines the tool structure,
[frontendToolsAtom](/docs/react-core/state-management/frontend-tools-atom#frontendtoolsatom) stores the registry, [useAgent](/docs/react-core/hooks/use-agent#useagent) orchestrates execution.

*Example*

```tsx
useAgentTool({
  name: 'get_clipboard',
  description: 'Read the current clipboard contents',
  inputSchema: z.object({}),
  execute: async () => {
    return await navigator.clipboard.readText();
  },
  render: ({ output }) => <code>{output}</code>,
});
```
