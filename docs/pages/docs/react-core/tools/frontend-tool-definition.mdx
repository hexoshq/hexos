Defines a tool that executes in the browser rather than on the server.

Frontend tools have access to browser APIs (clipboard, geolocation, DOM, etc.)
and execute locally when the agent invokes them. The optional `render` function
provides custom UI for displaying the tool's input and output.

When the LLM calls a frontend tool, [useAgent](/docs/react-core/hooks/use-agent#useagent) detects the matching name in
[frontendToolsAtom](/docs/react-core/state-management/frontend-tools-atom#frontendtoolsatom) and executes it locally instead of waiting for a server result.

Related: [useAgentTool](/docs/react-core/hooks/use-agent-tool#useagenttool) registers these tools, [ToolDefinition](/docs/common/tools/tool-definition#tooldefinition) is the
server-side equivalent.

```ts
interface FrontendToolDefinition<TInput = unknown, TOutput = unknown> {
    name: string;
    description: string;
    inputSchema: z.ZodType<TInput>;
    requiresApproval?: boolean;
    execute?: (input: TInput) => Promise<TOutput>;
    render?: (props: { input: TInput; output?: TOutput; state: ToolCallState }) => React.ReactNode;
}
```
